# 加密密码

加密密码是 Friday 同步中最重要的安全凭证。本文解释它的作用和注意事项。

---

## 什么是加密密码？

加密密码是一个 **24 位的随机字符串**，在你首次激活 License 时自动生成。

示例格式：
```
Ab3Cd5Ef7Gh9Ij1Kl3Mn5Op7
```

---

## 它的作用

### 端到端加密

```
你的笔记 → [加密] → 云端存储 → [解密] → 其他设备
              ↑                      ↑
          使用密码               使用密码
```

| 阶段 | 说明 |
|------|------|
| 加密 | 笔记在上传前使用你的密码加密 |
| 存储 | 云端只存储加密后的数据 |
| 解密 | 下载时使用密码解密 |

### 安全保证

- **服务器无法查看**：即使服务器被攻破，攻击者也无法读取你的笔记
- **只有你能解密**：密码只存储在你的设备上，从不上传
- **强加密**：使用业界标准的加密算法

---

## 为什么必须保存它

{{% hint danger %}}
**密码丢失 = 数据丢失**

加密密码从不上传到服务器或存储在云端。  
如果你丢失了密码，**没有任何方法可以恢复你的云端数据**。
{{% /hint %}}

这是端到端加密的代价：

| 传统同步 | Friday 同步 |
|----------|-------------|
| 服务商可以重置你的密码 | 只有你能访问数据 |
| 服务商可以看到你的数据 | 服务商无法查看内容 |
| 安全性依赖服务商 | 安全性由你掌控 |

## 如何安全保存

### 推荐方法

{{< tabs "save-password" >}}
{{% tab "密码管理器" %}}

**最推荐**

使用 1Password、Bitwarden、KeePass 等：

1. 创建一个新条目，命名为 "Friday 加密密码"
2. 将密码粘贴到密码字段
3. 保存并同步到其他设备

好处：
- 加密存储
- 多设备访问
- 不会丢失

{{% /tab %}}
{{% tab "安全笔记" %}}

**替代方案**

保存在其他安全笔记应用中：

1. 使用 Apple Notes、Google Keep 等（有锁定功能）
2. 创建一个加锁的笔记
3. 写下密码

注意：确保笔记本身是加密/加锁的。

{{% /tab %}}
{{% tab "纸质备份" %}}

**离线备份**

对于极其重要的密码，纸质备份是最后的手段：

1. 在纸上清晰写下密码
2. 密封在信封中
3. 妥善保管（保险箱、上锁的柜子等）

注意：
- 书写清晰，避免混淆（0/O、1/l）
- 告诉你信任的人位置（以防万一）

{{% /tab %}}
{{< /tabs >}}

### 不推荐的方式

❌ 只保存在手机备忘录中（手机丢失就没了）  
❌ 发送到不安全的邮箱  
❌ 保存在未加密的文本文件中  
❌ 只靠记忆

## 如何查看当前密码

如果你已经激活但忘记保存：

1. 打开 **设置** → **Friday**
2. 找到「安全」区域
3. 点击「加密密码」旁边的 **显示密码** 按钮
4. 复制并保存到安全的地方

{{% hint warning %}}
**注意**  
密码只在你的设备上可见。如果你更换设备或重装 Obsidian，你需要之前保存的密码。
{{% /hint %}}

## 常见问题

### Q：能更改加密密码吗？

目前不支持。如果需要更改：
1. 导出/备份所有数据到本地
2. 重新创建同步配置
3. 使用新密码重新上传

### Q：密码丢失了怎么办？

如果密码确实丢失：

1. **本地数据没问题**：当前设备上的笔记不受影响
2. **云端数据无法恢复**：加密的云端数据无法解密
3. **重新开始**：可以用新的加密密码重新上传本地数据

### Q：所有设备的密码一样吗？

是的。所有设备使用 **相同的加密密码**，这样它们才能互相解密数据。

### Q：密码会过期吗？

不会。加密密码没有过期日期。只要你的 License 有效，密码就能使用。

## 安全总结

1. ✅ 激活后 **立即** 保存密码
2. ✅ 使用密码管理器
3. ✅ 做一个离线备份（纸质）
4. ✅ 不要和 License Key 放在同一个地方
5. ❌ 不要分享给他人（除非你信任他们查看你的笔记）
