# 第一台设备设置

这是你首次设置 Friday 同步的完整指南。请仔细阅读每个步骤。

---

## 前提条件

- ✅ 已安装 Friday 插件
- ✅ 已获取 License Key
- ✅ 网络连接正常

---

## 设置步骤

### 步骤 1：激活 License

1. 打开 **设置** → **Friday**
2. 在 License 区域输入你的 Key
3. 点击 **Activate** 按钮
4. 等待激活成功

### 步骤 2：保存加密密码 ⚠️

{{% hint danger %}}
**极其重要！**  
激活后，系统会自动生成一个 **24 位的加密密码**。  
这个密码用于加密你的所有数据。**丢失后无法恢复！**
{{% /hint %}}

激活后，在「安全」区域你会看到：

```
加密密码: ************************
         [显示密码] [复制]
```

**请立即执行以下操作：**

1. 点击 **显示密码** 按钮
2. **复制** 密码
3. 保存到安全的地方：
   - 密码管理器（推荐）
   - 写在纸上并妥善保管
   - 发送到自己的邮箱

### 步骤 3：上传本地数据到云端

保存好密码后：

1. 找到「同步」区域
2. 点击 **上传本地到云端** 按钮
3. 等待上传完成

上传过程中会显示进度，完成后有成功提示。

---

## 上传完成后

上传成功后，同步会自动开始：

- 状态栏显示同步图标
- 新建、修改、删除的文件自动同步到云端
- 你可以在其他设备上下载这些数据

---

## 验证同步状态

你可以通过以下方式确认同步正常工作：

1. **状态栏图标**：正常同步时显示 ✓
2. **日志信息**：打开开发者控制台查看详细日志
3. **测试**：修改一个文件，观察是否自动同步

---

## 关于加密

Friday 使用 **端到端加密**：

| 特性 | 说明 |
|------|------|
| 加密位置 | 数据在离开你的设备前就已加密 |
| 算法 | 使用强加密算法 |
| 密码存储 | 只存储在你的设备上，从不上传 |
| 安全性 | 即使服务器被攻破，数据也无法被解密 |

## 常见问题

### Q：上传很慢？

首次上传需要传输所有文件。时间取决于：
- 你的网络速度
- 仓库大小
- 图片和附件数量

建议在稳定网络环境下进行首次上传。

### Q：上传失败？

1. 检查网络连接
2. 再次点击「上传本地到云端」
3. 如果问题持续，尝试重启 Obsidian

### Q：忘记保存加密密码了？

{{% hint warning %}}
如果你还没有关闭设置页面，密码仍然显示在那里——现在就保存它！

如果已经关闭但没保存：
1. 你仍然可以在设置中看到加密密码
2. 点击「显示密码」查看并保存它
{{% /hint %}}
