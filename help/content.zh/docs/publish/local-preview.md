# 本地预览

发布之前，预览功能让你确保网站效果符合预期。

---

## 为什么要预览？

- ✅ **检查布局**：确保内容正确显示
- ✅ **验证链接**：确保内部链接正常工作
- ✅ **查看样式**：确认主题效果符合预期
- ✅ **测试功能**：检查导航、搜索等功能

---

## 生成预览

### 步骤

1. 在右侧面板完成内容和主题配置
2. 点击 **Generate Preview** 按钮
3. 等待构建完成（显示进度条）
4. 成功后，浏览器自动打开预览页面

### 构建过程

构建过程包括：

| 阶段 | 说明 |
|------|------|
| 1. 准备环境 | 创建临时目录，配置主题 |
| 2. 处理内容 | 转换 Markdown，处理图片 |
| 3. 构建站点 | 生成 HTML 页面 |
| 4. 启动服务器 | 启动本地预览服务器 |

---

## 预览链接

构建完成后，你会看到预览链接：

```
Preview Link
http://localhost:8090/
```

点击链接在浏览器中查看。

{{% hint info %}}
**关于端口**  
默认使用 8090 端口。如果被占用，系统会自动选择其他端口。
{{% /hint %}}

---

## 实时重载

Friday 支持 **实时重载**：

1. 保持预览页面打开
2. 修改你的笔记内容
3. 保存后，预览页面自动刷新

实时看到你的更改！

---

## 重新生成预览

如果你：
- 修改了内容
- 更换了主题
- 调整了配置

点击 **Regenerate Preview** 按钮来更新。

---

## 导出站点

预览满意后，你可以将网站导出为 ZIP 文件：

1. 点击 **Export Site** 按钮
2. 选择保存位置
3. 获得包含完整网站文件的 ZIP 压缩包

导出的 ZIP 可以：
- 上传到任何静态托管服务
- 部署到你自己的服务器
- 作为备份保存

---

## 预览目录结构

预览文件保存在插件目录中：

```
.obsidian/plugins/friday/preview/
└── {preview-id}/
    ├── config.json      # 站点配置
    ├── content/         # 内容文件
    └── public/          # 生成的网站文件
```

{{% hint warning %}}
**注意**  
预览文件是临时的，可能会被清理。使用导出功能来保留它们。
{{% /hint %}}

## 常见问题

### Q：预览构建失败？

检查：
1. 内容选择正确吗？
2. 主题选择了吗？
3. 阅读错误信息

常见原因：
- Markdown 语法错误
- 图片路径不正确
- 主题下载失败

### Q：预览页面打不开？

可能原因：
1. **端口被占用**：查看日志中的实际端口
2. **防火墙阻止**：允许 Obsidian 网络访问
3. **服务器崩溃**：重新生成预览

### Q：预览和发布后的效果不一样？

预览和发布使用相同的构建过程，效果应该一致。如果不同：
1. 检查 SitePath 配置
2. 确认使用的是最新预览
3. 清除浏览器缓存后检查

### Q：如何清理预览文件？

预览文件会占用磁盘空间。清理方法：
1. 打开项目管理（点击左侧边栏 Friday 图标）
2. 找到项目
3. 点击「清除历史」

或手动删除 `.obsidian/plugins/friday/preview/` 目录。

### Q：能同时预览多个站点吗？

可以。每个预览使用不同的端口，所以你可以：
1. 预览站点 A
2. 不关闭的情况下，预览站点 B
3. 两个预览同时可访问
