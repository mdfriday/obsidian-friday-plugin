# 项目管理

当你创建多个网站或长期维护一个网站时，项目管理可以帮助你高效管理所有配置。

---

## 什么是项目？

在 Friday 中，一个 **项目** 是一组网站配置的集合：

| 配置项 | 说明 |
|--------|------|
| 内容路径 | 网站内容的源文件夹/文件 |
| 站点名称 | 网站标题 |
| 主题 | 选择的主题 |
| SitePath | 网站路径配置 |
| 发布配置 | Netlify/FTP 账号信息 |
| 高级设置 | GA、Disqus、密码等 |

每次构建预览时，这些配置会自动保存为一个项目。

---

## Friday 按钮

在 Obsidian 的左侧边栏，你会看到一个 **🎲 骰子图标** ——这就是 Friday 按钮。

### 点击 Friday 按钮

会打开 **项目管理** 面板，显示：

- 所有已保存的项目列表
- 每个项目的基本信息
- 操作按钮

---

## 项目列表

项目列表显示每个项目的：

| 信息 | 说明 |
|------|------|
| **项目名称** | 你设置的站点名称 |
| **内容路径** | 内容来源 |
| **主题** | 选择的主题名称 |
| **更新时间** | 最后构建时间 |

---

## 项目操作

### 应用到面板

点击项目卡片或「应用」按钮，会将该项目的所有配置加载到右侧面板：

- ✅ 自动设置内容路径
- ✅ 自动设置站点名称
- ✅ 自动设置主题
- ✅ 自动设置发布配置
- ✅ 自动设置高级选项

这样你就可以 **一键恢复** 之前的配置，无需重新设置。

### 导出历史构建

如果项目有构建历史，你可以导出之前的构建结果：

1. 点击「导出」按钮
2. 选择保存位置
3. 获得包含网站文件的 ZIP 压缩包

### 清除历史

如果项目的预览文件占用太多空间：

1. 点击「清除历史」按钮
2. 确认清除
3. 该项目的所有预览文件被删除

{{% hint warning %}}
**注意**  
清除历史后，之前的预览文件无法恢复。如需保留，请先导出。
{{% /hint %}}

## 项目管理的好处

### 1. 快速切换项目

如果你维护多个网站：

- 博客
- 文档站点
- 作品集
- 公司官网

每个都有不同的配置。项目管理让你一键切换，无需记住每个站点的设置。

### 2. 配置不会丢失

所有配置自动保存：

- 主题选择
- 发布账号
- 高级设置

即使关闭 Obsidian，配置依然保留。

### 3. 历史追踪

查看每个项目的：

- 构建历史
- 发布记录
- 变更时间

## 项目存储位置

项目数据保存在插件目录中：

```
.obsidian/plugins/friday/projects-data.json
```

预览文件保存在：

```
.obsidian/plugins/friday/preview/{preview-id}/
```

## 常见问题

### Q：项目会自动保存吗？

是的。每次生成预览时，当前配置会自动保存为项目（或更新已有项目）。

### Q：如何删除项目？

目前不支持直接删除。你可以：
1. 清除该项目的历史（释放空间）
2. 忽略不需要的项目

### Q：项目太多怎么办？

项目列表按更新时间排序，最近使用的排在前面。常用项目自然会出现在顶部。

### Q：能手动编辑项目配置吗？

不推荐。项目配置会在每次构建时自动更新。如需修改：
1. 将项目应用到面板
2. 修改配置
3. 重新生成预览

### Q：项目能同步到其他设备吗？

目前项目数据是本地的，不会通过 Friday 同步。如果需要：
1. 手动备份 `projects-data.json` 文件
2. 或在每台设备上单独配置

## 工作流建议

### 单项目用户

如果你只维护一个网站：

1. 正常使用，项目会自动保存
2. 下次发布时，配置已经在那里
3. 不需要考虑项目管理

### 多项目用户

如果你维护多个网站：

1. 为每个网站使用不同的内容文件夹
2. 每个网站构建一次，配置自动保存
3. 切换时，点击 Friday 按钮
4. 选择项目，一键应用配置
5. 修改并重新构建/发布
